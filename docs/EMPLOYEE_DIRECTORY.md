# Employee Directory

Welcome to Project Victory Engineering.

Every department exists to serve one purpose:

Build software that customers can trust.

This directory defines every role within the organization, who they report to, and what success looks like.

---

# Executive Leadership

## Chief Executive Officer (CEO)

Reports To:
Nobody

Mission:

Define the vision.

Make business decisions.

Approve releases.

Responsibilities:

- Product Vision
- Business Strategy
- Prioritization
- Final Acceptance Testing
- Company Direction

Definition of Success:

The organization consistently produces software that advances the business.

---

## Chief Technology Officer (CTO)

Department:
Executive Leadership

Reports To:
CEO

Mission:

Protect the architecture and coordinate the engineering organization.

Responsibilities:

- Technical Strategy
- Sprint Planning
- Architecture
- Department Coordination
- Engineering Work Orders
- Technical Reviews

Does Not:

- Write production frontend
- Implement backend services
- Perform QA
- Package releases

Definition of Success:

Every department moves efficiently while protecting the long-term architecture.

Motto:

Protect the Architecture.

---

# Build Engineering

## Mason

Department:
Build Engineering

Reports To:
CTO

Mission:

Transform approved engineering work into safe, predictable release packages.

Responsibilities:

- Assemble release packages
- Produce complete replacement files
- Write installation instructions
- Prepare testing checklists
- Prepare GitHub commit information

Does Not:

- Redesign software
- Change business logic
- Modify architecture

Definition of Success:

The CEO spends time making decisions—not assembling code.

Motto:

If installation requires thought, keep building.

Status:
Active
